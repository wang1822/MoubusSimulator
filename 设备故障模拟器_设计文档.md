## 1. 项目概述

### 1.1 目标

开发一个 **Modbus Slave 设备模拟器**，用于对接 GS215 EMS（能源管理系统）进行功能联调测试。模拟器可在无实物设备的情况下，通过界面手动设置各设备的遥测数值、运行状态和告警/故障信息，供 EMS 系统通过 Modbus TCP/RTU 读取。

### 1.2 模拟设备类型

| 设备简称 | 中文全称 |
|---|---|
| PCS | 储能变流器 |
| BMS | 电池管理系统 |
| MPPT | 光伏 |
| STS | 转换开关 |
| 空调 | 空调 |
| 除湿机 | 除湿机 |
| 柴发 | 柴油发电机组 |
| 电表 | 三相多功能电表 |
| DI/DO | 动环控制器数字量 |
| 气体检测 | 气体检测仪 |

### 1.3 核心功能概览

| 功能 | 说明 |
|---|---|
| Modbus Slave | 支持 TCP（端口可配）和 RTU（串口可选），可配置从站地址 |
| 数据编辑 | 所有遥测字段通过输入框实时修改，修改后即刻写入内存寄存器 |
| 故障/告警注入 | 多选复选框（bitmask 型故障字）+ 下拉单选（互斥状态字段） |
| 配置管理 | 一键导出当前所有字段为 JSON 快照，支持下次加载恢复 |
| 日志监控 | 显示 EMS 发来的每条 Modbus 请求记录（时间/FC/地址/数量） |
| 异常处理 | 端口占用、串口不存在、输入非法等场景均有友好提示 |

```
┌───────────────────────────▼─────────────────────────────┐
│                  ViewModel 层                             │
│  MainViewModel（连接配置、启停、日志、导入导出）              │
│  ├── PcsViewModel                                         │
│  ├── BmsViewModel                                         │
│  ├── MpptViewModel                                        │
│  ├── StsViewModel                                         │
│  ├── AcViewModel（空调）                                   │
│  ├── DehumViewModel（除湿机）                              │
│  ├── GensetViewModel（柴发）                               │
│  ├── PowerMeterViewModel（电表）                           │
│  ├── DiDoViewModel（DI/DO）                                │
│  └── GasDetectorViewModel（气体检测）                      │
└───────────────────────────┬─────────────────────────────┘
                            │  调用服务接口（依赖注入）
┌───────────────────────────▼─────────────────────────────┐
│                  Service 层                               │
│  IModbusSlaveService  →  ModbusSlaveService（Slave 监听）  │
│  IRegisterMapService  →  RegisterMapService（字段→寄存器） │
│  RegisterBank（线程安全内存寄存器，65536 holding regs）      │
└─────────────────────────────────────────────────────────┘
```

### 3.2 数据流向

```
用户在 UI 修改字段值
    → ViewModel 属性触发 OnPropertyChanged
    → ViewModel 调用 RegisterMapService.Flush(device)
    → RegisterBank 内存寄存器更新
         ↑
EMS 发起 Modbus FC03 轮询
    → ModbusSlaveService 响应
    → 从 RegisterBank 读取对应地址范围返回给 EMS
    → 触发 OnRequest 事件 → MainViewModel 追加日志条目
```

### 3.3 线程模型

| 线程 | 职责 |
|---|---|
| UI 主线程 | XAML 渲染、用户输入、ViewModel 属性变更 |
| Modbus 监听线程（Task） | 异步等待 TCP 连接 / RTU 帧，处理请求，读取 RegisterBank |
| 日志派发（Dispatcher） | Modbus 线程回调通过 Dispatcher.BeginInvoke 更新 UI 日志 |

> `RegisterBank` 内部所有读写均持有 lock，保证线程安全。

## 6. 功能设计

### 6.1 Modbus Slave 功能

#### 6.1.1 支持模式

| 模式 | 参数 | 说明 |
|---|---|---|
| TCP | 端口（默认502）、Slave ID（默认1） | 支持多客户端并发连接 |
| RTU | 串口号（COМx）、波特率（4800/9600/19200/38400/115200）、Slave ID | 通过系统串口 |

#### 6.1.2 支持功能码

| FC | 说明 | 操作方向 |
|---|---|---|
| FC03 | 读保持寄存器（Read Holding Registers） | EMS → Slave（只读） |
| FC06 | 写单个寄存器（可选扩展） | EMS → Slave（控制下行） |
| FC16 | 写多个寄存器（可选扩展） | EMS → Slave（控制下行） |

#### 6.1.3 启停逻辑

1. 用户配置连接参数（TCP/RTU、端口/串口、Slave ID）
2. 点击"启动监听"按钮 → 服务层绑定端口或打开串口，开始异步监听
3. 状态指示灯变绿，按钮文本切换为"停止监听"
4. 点击"停止监听" → 关闭连接，释放资源

### 6.2 数据编辑功能

- 用户在各设备面板的 TextBox 中输入数值
- 输入框采用 `UpdateSourceTrigger=PropertyChanged`，实时绑定到 ViewModel 属性
- 属性变更触发 `RegisterMapService.Flush(device)`，将所有字段按协议规则写入内存 `RegisterBank`
- float32 字段（如电压/电流）拆分为高16位+低16位，按 Big-Endian 字序写入两个连续寄存器
- int16 字段（如温度/状态编码）直接写单个 ushort 寄存器

### 6.3 故障/告警注入功能

- **多选 CheckBox（bitmask 型）**：用户勾选多个告警项，ViewModel 将所有勾选项的 BitMask 按位 OR 合并后写入对应寄存器
- **下拉 ComboBox（枚举互斥型）**：用户选择一项，ViewModel 将枚举值直接写入对应寄存器
- 切换选项/勾选状态后，寄存器实时更新，EMS 下次轮询即可读到最新值
- 支持"一键清除本设备所有故障"快捷按钮，将该设备所有故障/告警字寄存器清零

### 6.4 配置管理功能

#### 导出配置（快照）
- 点击"导出配置"→ 弹出保存文件对话框（.json 格式）
- 将当前所有设备 ViewModel 的字段值序列化为 JSON 文件

#### 导入配置（恢复快照）
- 点击"导入配置"→ 弹出打开文件对话框
- 读取 JSON 文件，反序列化后更新所有设备 ViewModel 属性
- 各设备面板 UI 自动刷新，RegisterBank 同步写入

#### 快照 JSON 结构（示意）

```
{
  "SlaveId": 1,
  "Protocol": "TCP",
  "Port": 502,
  "Devices": {
    "PCS": { "DcVoltage": 750.0, "OperatingState": 0, "Fault1": 0, ... },
    "BMS": { "Soc": 80.0, "SystemState": 0, "Alarm1": 0, ... },
    ...
  }
}
```

### 6.5 异常处理设计

#### 6.5.1 异常分类与处理策略

| 异常类型 | 触发场景 | 处理方式 | 用户提示 |
|---|---|---|---|
| **端口占用** | TCP 端口已被其他进程占用 | 捕获 SocketException，停止启动流程 | 弹窗提示：「端口 502 已被占用，请更换端口后重试」 |
| **串口不存在** | 选择的 COMx 串口不存在 | 捕获 UnauthorizedAccessException / IOException | 弹窗提示：「串口 COM3 不存在或已被占用」 |
| **输入值非法** | TextBox 输入了非数字或超量程数值 | 属性 Setter 做类型校验，拒绝写入，高亮红框 | 字段边框变红 + 日志面板追加 WARN 行 |
| **输入值超量程** | 数值超出协议定义量程 | 自动截断到边界值，记录警告 | 日志追加「已自动截断至最大值 xxxxx」 |
| **JSON 快照格式错误** | 导入了格式不兼容的 JSON | 捕获 JsonException，中断加载 | 弹窗提示：「配置文件格式错误，请检查 JSON 内容」 |
| **快照字段缺失** | JSON 中缺少某设备字段 | 用默认初始值填充，记录到日志 | 日志追加「字段 BMS.Soc 缺失，使用默认值 80.0」 |
| **Modbus 通信异常** | 底层 NModbus4 抛出异常 | try-catch 包裹请求处理，记录 ERROR，不崩溃主程序 | 日志面板追加 ERROR 行 |
| **RegisterBank 并发冲突** | UI 线程和 Modbus 线程同时读写 | RegisterBank 内部 lock 保护，无需上层处理 | 无（透明处理） |
| **文件访问权限不足** | 导出路径无写权限 | 捕获 UnauthorizedAccessException | 弹窗提示：「无权限写入目标路径，请选择其他位置」 |
| **意外崩溃** | 未预期异常 | App.xaml.cs 注册全局 UnhandledException 处理 | 弹出"程序异常"对话框，显示简要信息，并将完整堆栈写入日志文件 |

#### 6.5.2 全局兜底机制

- **Application.DispatcherUnhandledException**：捕获 UI 线程未处理异常，记录日志，提示用户保存配置并重启
- **TaskScheduler.UnobservedTaskException**：捕获 Task 中未观察的异常（主要来自 Modbus 后台线程），记录后不崩溃
- 异常日志文件路径：`./logs/error-{yyyy-MM-dd}.log`

### 6.6 日志功能设计

#### 6.6.1 日志分级

| 级别 | 颜色 | 用途 |
|---|---|---|
| DEBUG | 灰色 | 开发调试信息（默认不显示，可通过配置开启） |
| INFO | 白色 | Modbus 请求记录、启停事件、配置导入导出 |
| WARN | 黄色 | 输入值截断、快照字段缺失、非致命异常 |
| ERROR | 红色 | Modbus 通信异常、文件操作失败 |
| FATAL | 深红色 | 程序崩溃捕获 |

#### 6.6.2 界面日志面板

- 位于连接配置区下方，固定高度约 100px，内置 ScrollViewer，自动滚动到最新一行
- 每条日志格式：`HH:mm:ss.fff  [LEVEL]  消息内容`
- Modbus 请求日志格式：`14:32:01.123  [INFO]  FC03  addr=7296  qty=345  slaveId=1  来源=192.168.1.100`
- 最多保留最近 500 条，超出自动清除最旧条目
- 提供"清空日志"按钮
- 提供"折叠/展开日志面板"按钮

#### 6.6.3 文件日志（NLog）

| 日志文件 | 内容 | 策略 |
|---|---|---|
| `logs/app-{yyyy-MM-dd}.log` | INFO 及以上全部日志（含 Modbus 请求） | 每日滚动，最多保留 30 天 |
| `logs/error-{yyyy-MM-dd}.log` | ERROR / FATAL 日志 + 完整异常堆栈 | 每日滚动，最多保留 90 天 |

#### 6.6.4 关键事件必须记录

| 事件 | 日志级别 |
|---|---|
| Slave 启动/停止 | INFO |
| 客户端连接/断开 | INFO |
| 每条 FC03/FC06 请求 | INFO |
| 输入值被截断 | WARN |
| JSON 快照导入成功/失败 | INFO / ERROR |
| 任何异常 | ERROR / FATAL |

## 7. 核心模块职责说明

### 7.1 数据模型层（Models）

| 类 | 职责 |
|---|---|
| `DeviceModelBase` | 定义所有设备共有属性：DeviceName、BaseAddress、SlaveId；声明抽象方法 `ToRegisters()` 和 `FromRegisters()` |
| `XxxModel`（各设备） | 持有该设备所有遥测字段的 CLR 属性；实现 `ToRegisters()` 将字段按协议字序打包为 ushort 数组 |
| `XxxFaultBits`（枚举） | 用 `[Flags]` 枚举声明故障字各 bit 含义，便于类型安全地做位运算 |

### 7.2 服务层（Services）

| 类/接口 | 职责 |
|---|---|
| `RegisterBank` | 65536 个 holding register 的内存池；所有读写加 lock；提供 ReadHolding、WriteHolding、WriteHoldingFloat32、ReadHoldingRange 方法 |
| `IModbusSlaveService` | 定义 StartTcp、StartRtu、Stop、IsRunning、Registers 属性，以及 OnRequest 事件 |
| `ModbusSlaveService` | 使用 NModbus4 创建 Slave，将 RegisterBank 适配为 ISlaveDataStore；异步监听 TCP 或 RTU；触发 OnRequest 事件供 ViewModel 订阅 |
| `IRegisterMapService` | 定义 Flush（模型→寄存器）、SaveSnapshot、LoadSnapshot |
| `RegisterMapService` | 调用各 Model 的 ToRegisters() 将结果写入 RegisterBank；负责 JSON 序列化/反序列化快照文件 |

### 7.3 ViewModel 层（ViewModels）

| 类 | 职责 |
|---|---|
| `MainViewModel` | 连接参数（IP/端口/串口/波特率/SlaveID）；启停命令；日志 ObservableCollection；导出/导入命令；持有10个设备 ViewModel 实例 |
| `DeviceViewModelBase` | IsExpanded（面板折叠状态）；持有 IRegisterMapService 引用；提供 FlushToRegisters() 供子类在属性变化时调用 |
| `XxxViewModel`（各设备） | 每个遥测字段对应一个 `[ObservableProperty]`；属性 Changed 时调用 FlushToRegisters；包含 ComboBox 的选项列表（`List<ComboItem>`）；包含告警多选列表（`ObservableCollection<AlarmItem>`） |
| `AlarmItem` | Label（中文名）+ BitMask（bit权重）+ IsChecked（双向绑定）；供 CheckBox 列表使用 |
| `ComboItem` | Value（int） + Display（中文显示名）；供 ComboBox ItemsSource 使用 |

### 9.2 寄存器写入规则

| 数据类型 | 占用寄存器数 | 写入方式 |
|---|---|---|
| uint16 / int16 | 1 | 直接写单个 ushort |
| uint32 / int32 | 2 | 高16位在低地址，低16位在高地址（Big-Endian） |
| float32 | 2 | IEEE 754，高字在低地址（AB CD 字序） |
| string（SN号等） | N | 每寄存器存2字节 ASCII |
| bitmask（告警字） | 1 | 多个 bit OR 合并后写 1 个 uint16 |

### 9.3 float32 字序说明

float32 占用两个连续寄存器，按 Big-Endian 字序（AB CD）：

```
内存字节顺序: [D] [C] [B] [A]（Little-Endian IEEE754）
寄存器 offset+0  ← (bytes[3] << 8) | bytes[2]   高字（AB）
寄存器 offset+1  ← (bytes[1] << 8) | bytes[0]   低字（CD）
```
