## 1. 项目概述

### 1.1 目标

开发一个 **Modbus 主/从站模拟器**，支持两种工作模式：

- **从站模式（Slave）**：模拟 GS215 EMS 下属设备（PCS/BMS/MPPT 等），供 EMS 系统通过 Modbus TCP/RTU 轮询，无需真实硬件即可完成联调测试。
- **主站模式（Master）**：主动连接并轮询目标设备（真实硬件或另一个模拟器实例），读写寄存器，用于调试和数据采集。

两种模式不同时运行，用户在界面上手动切换。

### 1.2 模拟设备类型（从站模式）

| 设备简称 | 中文全称 |
|---|---|
| PCS | 储能变流器 |
| BMS | 电池管理系统 |
| MPPT | 光伏最大功率点跟踪器 |
| STS | 静态转换开关 |
| 空调 | 机房精密空调 |
| 除湿机 | 除湿机 |
| 柴发 | 柴油发电机组 |
| 电表 | 三相多功能电表 |
| DI/DO | 动环控制器数字量 |
| 气体检测 | 多合一气体检测仪 |

### 1.3 核心功能概览

| 模式 | 功能 | 说明 |
|---|---|---|
| **从站** | Modbus Slave 监听 | 支持 TCP / RTU，端口/串口/SlaveID 可配 |
| **从站** | 数据编辑 | 遥测字段通过输入框实时修改，即刻写入内存寄存器 |
| **从站** | 故障/告警注入 | bitmask 多选复选框 + 互斥状态下拉单选 |
| **从站** | 配置管理 | 一键导出/导入 JSON 快照，恢复上次设置 |
| **从站** | 日志监控 | 显示每条 Modbus 请求记录（时间/FC/地址/数量） |
| **主站** | Modbus Master 轮询 | 支持 TCP / RTU，定时轮询目标从站 |
| **主站** | 手动读写寄存器 | 指定起始地址和数量，立即读取或写入 |
| **主站** | 轮询结果展示 | 实时显示读取的寄存器值和更新时间 |
| **通用** | 异常处理 | 端口占用、串口不存在、输入非法等友好提示 |

## 3. 整体架构

### 3.1 模式切换逻辑

```
MainWindow
  ├── [从站模式] Tab  →  SlavePanel
  └── [主站模式] Tab  →  MasterPanel

切换时：停止当前运行的服务 → 切换面板 → 用户手动启动新模式
```

### 3.2 分层架构

```
┌─────────────────────────────────────────────────────────┐
│                     View 层（XAML）                      │
│  MainWindow → TabControl → SlavePanel / MasterPanel     │
│  SlavePanel：各设备 DeviceExpander（遥测/状态/告警注入）  │
│  MasterPanel：连接配置 + 轮询结果表格 + 手动读写区        │
└───────────────────────────┬─────────────────────────────┘
                            │ 双向数据绑定
┌───────────────────────────▼─────────────────────────────┐
│                   ViewModel 层                           │
│  MainViewModel（模式切换、日志、导入导出）                 │
│  ├── SlaveViewModel（连接参数、启停）                     │
│  │   └── 10 × DeviceViewModel（各设备字段）              │
│  └── MasterViewModel（轮询配置、结果展示）                │
└───────────────────────────┬─────────────────────────────┘
                            │ 依赖注入
┌──────────────┬────────────▼────────────────────────────┐
│ ISlaveService│        IMasterService                   │
│ TCP / RTU    │        TCP / RTU                        │
│ 从站监听      │        主站轮询                          │
└──────┬───────┴──────────────┬──────────────────────────┘
       │                      │
┌──────▼──────────────────────▼──────────────────────────┐
│              RegisterBank（共享数据中心）                 │
│  线程安全 65536 × ushort 内存寄存器池                    │
│  Slave：响应请求时从此读取 → 返回给 Master               │
│  Master：轮询结果写入此处 → 刷新 UI                      │
└────────────────────────────────────────────────────────┘
```

### 3.3 数据流向

**从站模式：**
```
用户修改字段值 → DeviceViewModel 属性变更
    → FlushToRegisters() → RegisterBank 更新
         ↑
EMS 发起 FC03 轮询 → SlaveService 从 RegisterBank 读取 → 返回响应
    → 触发 OnRequest 事件 → MainViewModel 追加日志
```

**主站模式：**
```
定时器触发 / 用户点击"立即轮询" → MasterService 发起 FC03 请求
    → 目标从站返回数据 → 写入 RegisterBank → MasterViewModel 刷新结果表格
    → 触发 OnResponse 事件 → MainViewModel 追加日志
```

### 3.4 线程模型

| 线程 | 职责 |
|---|---|
| UI 主线程 | XAML 渲染、用户输入、ViewModel 属性变更 |
| Slave 监听线程（Task） | 异步等待 TCP 连接/RTU 帧，处理请求，读 RegisterBank |
| Master 轮询线程（Task） | PeriodicTimer 定时触发，发起读写请求，写 RegisterBank |
| 日志派发（Dispatcher） | 后台线程回调通过 Dispatcher.BeginInvoke 更新 UI 日志 |

> `RegisterBank` 内部所有读写均持有 `lock`，保证线程安全。

## 6. 功能设计

### 6.1 模式切换

1. 用户点击顶部 RadioButton 切换模式
2. `MainViewModel` 检测到 `CurrentMode` 变更
3. 自动调用当前运行服务的 `Stop()`
4. 切换 ContentPresenter 显示对应 Panel
5. 用户在新面板手动点击启动

### 6.2 从站（Slave）功能

#### 支持模式

| 模式 | 参数 |
|---|---|
| TCP | 监听地址、端口（默认 502）、SlaveID（默认 1） |
| RTU | 串口号、波特率（4800/9600/19200/38400/115200）、SlaveID |

#### 支持功能码

| FC | 说明 |
|---|---|
| FC03 | 读保持寄存器（主要）|
| FC06 | 写单个寄存器（可选扩展）|
| FC16 | 写多个寄存器（可选扩展）|

#### 数据编辑

- TextBox 输入数值 → ViewModel 属性变更 → `FlushToRegisters()` → RegisterBank 更新
- float32 字段：拆分为高/低 16 位，按 Big-Endian 字序（AB CD）写入两个连续寄存器
- int16 字段：直接写单个 ushort 寄存器

#### 故障/告警注入

- **多选 CheckBox（bitmask）**：勾选多项 → 按位 OR 合并 → 写入告警字寄存器
- **下拉 ComboBox（互斥枚举）**：选择一项 → 直接写入状态寄存器
- **一键清除**：将该设备所有故障/告警字寄存器清零

#### 配置管理

- **导出**：所有设备 ViewModel 字段值序列化为 JSON 文件
- **导入**：读取 JSON → 反序列化 → 更新所有 ViewModel → RegisterBank 同步写入

### 6.3 主站（Master）功能

#### 支持模式

| 模式 | 参数 |
|---|---|
| TCP | 目标 IP、端口、SlaveID |
| RTU | 串口号、波特率、SlaveID |

#### 定时轮询

- `PeriodicTimer` 按配置间隔（默认 1000ms）触发
- 读取目标从站的 Holding Register
- 结果写入 RegisterBank，刷新结果表格
- 连接失败自动重连（指数退避，最大 30s）

#### 手动读写

- 用户指定起始地址 + 数量 → 立即发起 FC03 读取
- 用户指定地址 + 值 → 立即发起 FC06 写入

### 6.4 异常处理

| 场景 | 处理方式 |
|---|---|
| 端口/串口占用 | 捕获异常，弹窗提示，停止启动流程 |
| 输入值非法 | 拒绝写入，字段边框变红，日志追加 WARN |
| 输入值超量程 | 自动截断到边界值，日志追加 WARN |
| JSON 格式错误 | 中断加载，弹窗提示 |
| Modbus 通信异常 | 捕获异常，日志追加 ERROR，不崩溃主程序 |
| 意外崩溃 | 全局 UnhandledException，弹窗 + 写入错误日志 |

### 6.5 日志功能

| 级别 | 颜色 | 用途 |
|---|---|---|
| INFO | 白色 | 请求记录、启停事件、配置导入导出 |
| WARN | 黄色 | 输入值截断、非致命异常 |
| ERROR | 红色 | 通信异常、文件操作失败 |

- 界面日志最多保留 500 条，自动滚动到最新
- 文件日志：`logs/app-yyyy-MM-dd.log`（保留 30 天）

## 7. 核心模块职责说明

### RegisterBank

- 65536 个 Holding Register 内存池
- 所有读写加 `lock`，线程安全
- 提供 `Read` / `Write` / `WriteFloat32` / `ReadRange` 方法
- 从站和主站共用同一实例

### FloatRegisterHelper

- float32 ↔ 两个 uint16 互转
- 统一采用 AB CD（Big-Endian）字序

### ISlaveService / TcpSlaveService / RtuSlaveService

- 统一接口：`Start()` / `Stop()` / `IsRunning`
- TCP：基于 `TcpListener` + NModbus4 `ModbusTcpSlaveNetwork`
- RTU：基于 `SerialPort` + NModbus4 `ModbusSerialSlave`
- DataStore 与 RegisterBank 双向同步
- 触发 `OnRequest` 事件供 ViewModel 订阅日志

### IMasterService / TcpMasterService / RtuMasterService

- 统一接口：`Start()` / `Stop()` / `PollOnce()` / `IsRunning` / `PollInterval`
- TCP：基于 `TcpClient` + NModbus4 `ModbusMaster`
- RTU：基于 `SerialPort` + NModbus4 `ModbusSerialMaster`
- 轮询结果写入 RegisterBank
- 触发 `OnResponse` 事件供 ViewModel 刷新 UI

### IRegisterMapService / RegisterMapService

- `Flush(device)`：调用 `DeviceModel.ToRegisters()` 写入 RegisterBank
- `SaveSnapshot(path)`：序列化所有设备字段为 JSON
- `LoadSnapshot(path)`：反序列化 JSON → 更新 ViewModel → Flush

### DeviceViewModelBase

- `IsExpanded`：Expander 折叠状态
- `FlushToRegisters()`：同步 VM 属性 → Model → RegisterBank
- 子类通过 `[ObservableProperty]` + `OnXxxChanged` → `FlushToRegisters()` 实现实时同步

### MainViewModel

- 持有 `CurrentMode`（Slave/Master），切换时自动 Stop 旧服务
- 持有 `SlaveViewModel` + `MasterViewModel`
- 维护 `LogEntries`（ObservableCollection，最多 500 条）
- 处理导出/导入配置命令

### 9.2 数据类型写入规则

| 数据类型 | 占用寄存器数 | 写入方式 | 示例 |
|---|---|---|---|
| uint16 / int16 | 1 | 直接写入单个 ushort | `bank.Write(addr, (ushort)value)` |
| uint8 | 1 | 低8位有效，高8位清零 | `bank.Write(addr, (ushort)(value & 0xFF))` |
| float32 | 2 | AB CD 大端字序，高字写低地址 | `FloatRegisterHelper.ToRegisters(f)` → `[high, low]` |
| uint32 / int32 | 2 | 高16位写低地址，低16位写高地址 | `bank.Write(addr, (ushort)(val>>16)); bank.Write(addr+1, (ushort)val)` |
| uint64 / int64 | 4 | 同上，分4个寄存器 | 依次写高→低 |
| bitmask (uint16) | 1 | CheckBox IsChecked OR合并 | `items.Where(x=>x.IsChecked).Aggregate(0, (a,x)=>a|x.BitMask)` |
| bitmask (uint32) | 2 | 同上，拆为高/低16位 | `(uint)combined` → 高/低字分写 |

### 9.3 float32 字序说明

```
物理值 3.14f (IEEE 754: 0x4048F5C3)

写入方式（AB CD 大端）：
  addr+0 = 0x4048  (高16位，字节A=0x40，B=0x48)
  addr+1 = 0xF5C3  (低16位，字节C=0xF5，D=0xC3)

FloatRegisterHelper.ToRegisters(3.14f) → [0x4048, 0xF5C3]
```

### 9.4 设备索引偏移计算

```csharp
// PCS 第 n 台（n从0开始）起始地址
int pcsBase = 7296 + n * 512;

// BMS 第 n 台起始地址（单台128寄存器）
int bmsBase = 23680 + n * 128;

// MPPT 第 n 台起始地址（单台256寄存器）
int mpptBase = 40064 + n * 256;

// 空调 第 n 台起始地址（单台128寄存器）
int acBase = 52352 + n * 128;
```
