## 一、人员说明

| 成员 | 角色 | 背景 | Claude Code 使用重点 |
|---|---|---|---|
| **小王** | 项目负责人 / 架构师 | C# 有经验，主导开发 | 审查 PR、编写复杂逻辑、把控代码质量 |
| **小朱** | 功能开发 / UI 实现 | 新入职 | 用 `/modbus-simulator` 生成代码、用 `/git-commit` 提交、遇到问题直接问 Claude |

### 小朱的 Claude Code 使用说明

> 没有 C# 经验不是障碍，Claude 是你的结对编程伙伴。每次开发一个设备前，先做以下三件事：

```
第一步：打开 Claude Code，输入：
"帮我写 [设备名] 的完整代码，包括 Model、ViewModel 和 XAML 面板"

第二步：Claude 会输出完整代码 + 说明书，按说明书放入对应目录

第三步：完成后使用 /git-commit 提交，提交信息 Claude 自动生成
```

### Phase 3：设备面板实现（小朱 — 每个设备用 Claude 生成）

> 每个设备的开发流程完全一致，Claude 会处理所有 C# 细节。

**标准工作流：**

```
1. 告诉 Claude："帮我实现 [设备名]，起始地址 [xxx]，参考设计文档第8节"
2. Claude 输出 Model.cs + FaultBits.cs（如有）+ ViewModel.cs + Panel.xaml
3. 把文件放到对应目录（见设计文档第4节目录结构）
4. dotnet build 验证编译通过
5. /git-commit 提交
```

| # | 设备 | 起始地址 | 复杂度 | 关键说明 |
|---|---|---|---|---|
| 3.1 | STS 转换开关 | 仪表 1408 / 控制IO卡 1920 | ⭐⭐⭐ | 两套寄存器，注意区分仪表读数和控制信号 |
| 3.2 | 空调 | 52352 | ⭐⭐ | 温湿度 + 运行状态，字段较少 |
| 3.3 | 除湿机 | 53248 | ⭐ | 最简单，32个寄存器，适合练手 **（建议从这个开始）** |
| 3.4 | 柴发 | 53504 | ⭐⭐ | 电压/电流/频率 + 启停状态，含故障字 |
| 3.5 | DI/DO 动环控制器 | 60544 | ⭐⭐ | 数字量 0/1，ComboBox 控件为主，无 float32 |
| 3.6 | 气体检测 | 53760 | ⭐ | 16个寄存器，浓度值 + 告警阈值，字段少 |

> **建议顺序**：除湿机 → 气体检测 → 空调 → 柴发 → DI/DO → STS

## 四、PR 合并规则

> 参见 `GIT_WORKFLOW.md`，关键规则如下：

- 小朱每完成一个设备或控件，创建功能分支，提 PR 到 main
- 小王审查通过后合并，禁止直接推送 main
- 每个 PR 必须通过 `dotnet build`（`/git-commit` 自动检查）
- PR 标题格式：`feat(BMS): 添加 BMS 完整设备面板`

## 六、常见问题

**Q：小朱遇到编译错误怎么办？**

> 把报错信息完整粘贴给 Claude，说"这个错误怎么修"。Claude 会定位并修复。

**Q：小朱不知道某个字段的寄存器地址和比例系数怎么找？**
> 设计文档第 8-9 节有完整定义。告诉 Claude "查一下 BMS 的 SOC 字段寄存器地址"。

**Q：小朱写完代码不确定正不正确怎么办？**

> 提 PR 后通知小王用 `/review-pr` 审查，Claude 会指出问题。

**Q：两人同时修改了同一个文件冲突了怎么办？**

> 参见 `GIT_WORKFLOW.md` 冲突解决章节，或直接问 Claude "我有合并冲突，怎么解决"。
